# フロー A（確認画面あり）

確認画面でスパム判定を行い、完了処理では再スキャンしないパターンです。

## ファイル構成

```
flow-a-confirm/
  form.html      — 問い合わせフォーム（JS でトークン取得）
  confirm.php    — 確認画面・スパム判定・セッション保存
  submit.php     — 完了処理（セッションのスコアを使用）
```

## セットアップ

`confirm.php` の設定値を編集する：

```php
define('MAIL_TO', 'admin@example.com');
```

`form.html` の `YOUR_SITE_KEY` を書き換える。

## 動作の流れ

1. ユーザーがフォームを送信
2. JS がトークンを取得してサーバーへ POST
3. `confirm.php` がトークンで `/v1/analyze` を呼びスコアをセッションに保存
   - スコア 70 以上 → ブロック（確認画面を表示しない）
4. 確認画面を表示（内容確認・送信ボタン）
5. `submit.php` はセッションのスコアを使って処理（再スキャンなし）
   - セッションなし → 確認画面をすっ飛ばした直接送信としてエラー

## ポイント

- トークンは確認画面で消費されるため、完了処理に引き回す必要がない
- セッションの有無で確認画面通過を担保できる
